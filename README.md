# Vibe Coding Starter

一个面向 Codex 的起始仓库，用来快速开始你的 vibe coding 工作流。

这个仓库保留了最常用的一组 repo-local skills，放在 `.agents/skills/` 下。你可以直接 fork、clone，或者在此基础上继续裁剪成自己的工程模板。目标很简单：少折腾环境，多把精力放在写代码、调试、检查和交付上。

## 适合做什么

- 作为新的 Codex 项目模板
- 作为团队内部的 vibe coding 基线仓库
- 作为个人技能库和项目规范的起点

## 仓库内容

- `.agents/skills/`：可直接被 Codex 使用的本地技能
- `AGENTS-FastAPI.md`：FastAPI 场景下的协作约束示例
- `AGENTS-Python.md`：Python 项目的协作约束示例

## 内置技能

| Skill | 什么时候用 | 作用 |
| :--- | :--- | :--- |
| `/think` | 在写任何新功能、设计方案或架构决策之前 | 把粗略想法整理成结构清晰、经过验证、可批准的方案，不适合 bug 修复或小改动。 |
| `/design` | 在做 UI、组件、页面或视觉界面时 | 产出有明确审美方向的设计，而不是通用默认风格，不适合后端逻辑或数据流程。 |
| `/check` | 在完成实现后或合并前 | 审查 diff，自动修复安全的小问题，并在大改动时引入安全和架构视角复核，不适合用来探索思路或排查 bug。 |
| `/hunt` | 在排查报错、崩溃、异常行为或失败测试时 | 先定位根因，再决定修复方式，不适合代码评审或新功能开发。 |
| `/write` | 在明确需要写作、编辑或润色中英文文案时 | 去掉 AI 味，重写成更自然的表达，不适合代码注释、提交信息或行内文档。 |
| `/learn` | 在深入陌生领域、准备研究文章或整理资料成可发布内容时 | 按收集、消化、提纲、展开、润色、发布六个阶段推进，不适合快速查询或单文件阅读。 |
| `/read` | 在需要读取 URL、网页链接或 PDF 时 | 按平台路由抓取并转换成干净的 Markdown，不适合读取仓库里已经存在的本地文件。 |
| `/health` | 在 Codex 不听指令、行为异常、工具权限或 MCP 看起来不对时 | 审查当前项目的协作配置，并按严重程度标出问题，不适合调试代码或做 PR review。 |

## 使用方式

```bash
git clone https://github.com/hczs/vibe-coding-starter.git
cd vibe-coding-starter
```

然后按你的项目需要修改：

- 保留需要的 skills
- 删除不需要的示例文件
- 补充你自己的 `AGENTS.md` 或其他仓库级规则

如果你准备把它当成新项目起点，最稳妥的做法是直接在这个仓库上继续开发，或者复制内容后重新初始化成你自己的仓库。
